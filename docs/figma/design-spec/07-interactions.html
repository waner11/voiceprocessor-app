Animation Guidelines

Rules for applying motion consistently across the product.

Do

  • Use motion to provide feedback (button press, focus, success)
  • Keep transitions under 500ms
  • Use the shared easing curve for consistency
  • Animate properties that are GPU-accelerated (transform, opacity)
  • Respect prefers-reduced-motion
  • Use subtle elevation changes on hover for interactive elements

Don't

  • Use bounce or spring physics (feels playful, not premium)
  • Animate layout properties (width, height, margin, padding)
  • Stack multiple simultaneous animations
  • Add motion for decorative purposes only
  • Use delays longer than 100ms on UI feedback
  • Animate anything during scrolling

Accessibility: Reduced Motion

When prefers-reduced-motion is set, all transitions reduce to instant or minimal fade.

@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
